深度优先算法

  • 基本思想

    从一个顶点V0开始，沿着一条路一直走到底，如果发现不能到达目标解，那就返回到上一个节点，然后从另一条路开始走到底，这种尽量往深处走的概念即是深度优先的概念。

    为了实现深度优先搜索，首先选择一个起始顶点并需要遵守三个规则：

    1. 如果可能，访问一个邻接的未访问顶点，标记它，并把它放入栈中。
    2. 当不能执行规则1时，如果栈不空，就从栈中弹出一个顶点。
    3. 如果不能执行规则1和规则2，就完成了整个搜索过程。

  • 基本模型（参考《啊哈算法》，通过递归完成）

    算法关键在于解决“当前该怎么做”，以后的步骤应该和当前一致，通常通过for循环遍历将每种可能都尝试一遍，解决这一步之后变递归进入下一步。

广度优先算法

  • 基本思想

    它的思想是从一个顶点V0开始，辐射状地优先遍历其周围较广的区域。搜索过程：先访问节点v；再依次访问与v相邻的节点；访问这些节点之后，再访问与之相邻的节点。与深度优先搜索不同，其先遍历周围的是周围节点。常用于寻找最短路径的解决方案。

  • 基本模型（通过队列完成）

    算法的关键在于先将起始点入队，然后遍历周围的节点，已广度为第一优先级，所以通过广度优先算法最先找到的一定是路径最短的那个。

树的基本知识

  • 树的基本特性（树是一种特殊的图）

    1. 一棵树中的任意两个节点有且仅有唯一的一条路径联通

    2. 一棵树如果有n个节点，那么它一定有n-1条边

    3. 一棵树中加一条边会构成一个回路

  • 树的基本名词

    • 结点的度（Degree）：结点的子树个数；
    • 树的度：树的所有结点中最大的度数；
    • 叶结点（Leaf）：度为0的结点；
    • 父结点（Parent）：有子树的结点是其子树的根节点的父结点；
    • 子结点/孩子结点（Child）：若A结点是B结点的父结点，则称B结点是A结点的子结点；
    • 兄弟结点（Sibling）：具有同一个父结点的各结点彼此是兄弟结点；
    • 路径和路径长度：从结点n1到nk的路径为一个结点序列n1，n2，…，nk。ni是ni+1的父结点。路径所包含边的个数为路径的长度；
    • 祖先结点（Ancestor）：沿树根到某一结点路径上的所有结点都是这个结点的祖先结点；
    • 子孙结点（Descendant）：某一结点的子树中的所有结点是这个结点的子孙；
    • 结点的层次（Level）：规定根结点在1层，其他任一结点的层数是其父结点的层数加1；
    • 树的深度（Depth）：树中所有结点中的最大层次是这棵树的深度；

  • 树的应用

    树在现实生活中有很多应用，例如比赛的晋级图，家族的家谱图，操作系统的文件结构系统等都是树的实际应用。

二叉树

二叉树是一种特殊的树。其最大的特点是每个节点最多有两个儿子，其严格定义为：二叉树要么为空，要么由根节点，左子树和右子树构成。

  • 满二叉树与完全二叉树

    • 满二叉树：如果二叉树中每隔内部的节点都有两个儿子，这样的树叫做满二叉树。

    • 完全二叉树：除了最右边位置上有一个或几个叶节点缺少外，其他都是丰满的，这样的树称为完全二叉树。

  • 堆

    堆是一种特殊的树，如果所有父节点都比子节点小，符合这样的完全二叉树成为最小堆。相反，如果所有父节点都比子节点大，则称为最大堆。

散列

散列是最有用的数据结构之一。被用于快速查找某个目标。一般查找的方式例如遍历查找（运行时间为O（n））,二分查找（运行时间为O（logN）），而通过散列查找的速度可以达到O（1）；

  • 散列函数

    无论你给它什么数据，返回的都是一个数字。但并不是随机返回，而是根据数据返回对应的数字：

    • 它必须是一致的。例如，假设你输入apple时得到的是4，那么每次输入apple时，得到的都

    必须为4。如果不是这样，散列表将毫无用处。

    • 它应将不同的输入映射到不同的数字。 例如， 如果一个散列函数不管输入是什么都返回1，

    它就不是好的散列函数。最理想的情况是，将不同的输入映射到不同的数字。

    具体实现：通过将数据存放在返回数字对应的数组中，那么下次查找就可以直接再次通过计算数字很快找到数据存储的位置。（通过计算”Apple”返回数字3，则将apple的价格放在数组下标3中）

    ​

    ​

  • 散列的适用场景

    • 模拟映射关系；（例如：python中的dict，java中的map）
  • 防止重复；

  • 缓存/记住数据，以免服务器再通过处理来生成它们。

  • 冲突

    散列的原理是通过将数据通过散列函数计算得到一个数字实现的，自然而然我们就会想到如果两个数据通过散列函数得到的数字一样该怎么办呢？直接存储的话后面的数字就会将前面的数据覆盖。

  • 冲突的解决

    解决冲突最简单的办法就是在数组中不要直接存储我们要存储的数据，而是存储一个链表。实际的数据存储在这个链表里面。

    这样就不用担心冲突的问题了，虽然通过散列没有直接找到目标，但链表上的数据一般不多，比直接搜索也快多了。^-^。

Docker简介

​ Docker是一个装应用的容器，每个容器中运行着一个或多个应用，不同的容器之间相互隔离，而且容器之间是可以相互通信的。

​ Doker容器除了运行其中的应用之外，基本不消耗额外的系统资源保证应用性能的同时，尽量减少系统的开销。最厉害的是，即使其中某个崩溃了，其他应用也不会受到影响可以正常运行。

Docker的核心概念

  1. 镜像

    Docker镜像类似与虚拟机的镜像，一个镜像可以包含一个完整的操作系统，里面包含了我们应用以及必须的运行环境。镜像是docker的基础，它是一个只读的模版，我们可以直接下载很多镜像作为我们的模版。

  2. 容器

    Docker 容器和文件夹很类似，一个Docker容器包含了所有的某个应用运行所需要的环境。每一个 Docker 容器都是从 Docker 镜像创建的。Docker 容器可以运行、开始、停止、移动和删除。每一个 Docker 容器都是独立和安全的应用平台，Docker 容器是 Docker 的运行部分。

  3. 仓库

    Docker仓库类似于代码仓库，是Docker集中存放镜像文件的地方。通过不同的仓库，我们可以直接下载到很多现成的镜像。

##namenode元数据管理机制

[TOC]

客户端下载hdfs中文件的过程

  1. 客户端首先向namenode发送下载请求，然后nodenode返回请求文件的元信息给客户端（元数据包括文件切片后的每一块放在哪一个服务器上的信息）
  2. 客户端通过返回的元数据请求对应的databode，通过建立NIO连接管道传输数据。（如果文件被切分为多个切片则会分别请求三个datanode，然后在客户端合并。）

namenode管理元数据的机制

  1. 当元数据更新之后，首先在namenode的内存中更新，然后进行追加日志的操作。（日志记录了对文件的具体操作信息）。

  2. 每隔一段时间，通过Secondary NameNode每隔一段时间向namenode请求是否需要做需要做日志归档。如果需要，则namenode会滚动日志文件（创建新的文件继续写日志），已经写入的日志文件会被Secondary NameNode下载并写为镜像文件。

  3. 每一个block占元数据150byte，不管文件大小都会占用这么多，所以hdfs不适合存储大量小文件。
